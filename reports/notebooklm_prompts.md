# NotebookLM PPT 제작 프롬프트 가이드

> **사용 방법**  
> 1. NotebookLM에 `presentation_outline.md` 파일을 소스로 업로드  
> 2. 아래 프롬프트를 순서대로 입력해 각 슬라이드 내용 생성  
> 3. 생성된 텍스트를 PPT 도구(PowerPoint, Google Slides, Canva 등)에 붙여넣기

---

## STEP 0. 시작 전 (소스 업로드 후 첫 프롬프트)

```
이 문서는 GE(제너럴 일렉트릭) 주가 예측 머신러닝 프로젝트의 전체 설명서야.
이 내용을 바탕으로 대학 수업 발표용 PPT를 만들 거야.
슬라이드는 총 12장으로 구성하고, 각 슬라이드마다 제목, 핵심 내용(3줄 이내 불릿), 발표 멘트(30초 분량)를 만들어줘.
전체 발표 시간은 10분이야.
```

---

## STEP 1. 표지 슬라이드

```
PPT 표지 슬라이드를 만들어줘.
- 프로젝트 제목 (임팩트 있게)
- 부제목 (한 줄 요약)
- 발표자 이름 자리 표시
- 날짜
표지에 들어갈 한 줄 슬로건도 제안해줘.
```

---

## STEP 2. 프로젝트 목적 슬라이드

```
"왜 이 프로젝트를 했는가"를 설명하는 슬라이드를 만들어줘.
- 주식 예측을 머신러닝으로 접근한 이유
- 단순 분류(상승/하락)가 아닌 회귀(종가 예측)를 선택한 이유
- 자동화(n8n + Discord)까지 연결한 이유
청중이 "아, 이게 왜 의미 있는 프로젝트구나"를 느낄 수 있게 작성해줘.
```

---

## STEP 3. 전체 아키텍처 슬라이드

```
전체 시스템 흐름을 설명하는 슬라이드를 만들어줘.
데이터 수집 → 전처리 → 학습 → 예측 → API → 자동화 → Discord 알림까지
각 단계를 화살표로 연결한 텍스트 다이어그램과 각 단계의 핵심 설명을 포함해줘.
PPT에 넣을 수 있는 텍스트 박스 형식으로 만들어줘.
```

---

## STEP 4. 데이터 수집/전처리 슬라이드

```
데이터 수집과 전처리 슬라이드를 만들어줘.
- 어디서 데이터를 가져왔는지 (Yahoo Finance, yfinance)
- 얼마나 많은 데이터인지 (3,096행, 2014~현재)
- 어떤 기술적 지표를 만들었는지 (RSI, MACD, 볼린저밴드 등 28개 피처)
- 왜 이 지표들이 의미 있는지 간단히 설명
발표자가 비전공자 청중에게도 이해시킬 수 있는 쉬운 설명으로 작성해줘.
```

---

## STEP 5. 모델 구조 슬라이드

```
사용한 두 가지 모델(Dense MLP, LSTM-window MLP)을 설명하는 슬라이드를 만들어줘.
- 각 모델의 구조를 텍스트로 시각화 (입력→은닉층→출력)
- Dense 모델과 LSTM-window 모델의 차이점
- scikit-learn MLPRegressor를 선택한 이유 (macOS 호환성 문제로 TF 대체)
기술적이지만 너무 어렵지 않게, 핵심만 짚어줘.
```

---

## STEP 6. 모델 성능 결과 슬라이드

```
모델 평가 결과 슬라이드를 만들어줘.
실제 수치를 포함해:
- Baseline (MAE: 3.02, RMSE: 4.44, MAPE: 1.40%)
- Dense MLP (MAE: 6.82, RMSE: 10.21, MAPE: 2.70%)
- LSTM-window (MAE: 4.77, RMSE: 7.19, MAPE: 2.03%)

Baseline보다 모델이 낮은 이유를 솔직하게 설명하되,
"그래서 이 프로젝트가 실패한 게 아니라 현실적인 결론을 도출했다"는 맥락으로 작성해줘.
```

---

## STEP 7. 테마주 비교 분석 슬라이드

```
GE와 관련 테마주 비교 분석 슬라이드를 만들어줘.
비교 종목: GE Aerospace, GE Vernova(GEV), GE HealthCare(GEHC), RTX, Honeywell, Boeing
포함할 내용:
- 왜 이 종목들을 비교했는지 (GE 분사 역사, 항공우주 섹터 경쟁)
- 최근 성과 비교 (GEV가 +219% 1년 수익률로 압도적)
- 상관관계 분석 결과 (RTX가 GE와 가장 높은 상관관계)
인사이트 중심으로, 단순 표 나열이 아니라 "이런 의미가 있다"는 해석 포함해줘.
```

---

## STEP 8. n8n 자동화 슬라이드

```
n8n 자동화 워크플로우 슬라이드를 만들어줘.
- 매일 오전 7시에 자동 실행되는 스케줄러
- FastAPI(/predict, /compare)를 HTTP로 호출하는 방식 (executeCommand 미사용 이유 포함)
- Discord로 두 가지 메시지(예측 + 테마주 비교)를 자동 발송
- Google Sheets에 누적 기록

"코드 한 줄 안 짜고도 매일 리포트를 받을 수 있다"는 자동화의 가치를 강조해줘.
```

---

## STEP 9. Discord 알림 슬라이드

```
Discord 알림 결과물 슬라이드를 만들어줘.
실제 Discord에 발송되는 메시지 형식을 텍스트로 보여주면서
- 예측 메시지에 포함된 정보 (현재가, 예측가, 수익률, RSI, 변동성)
- 테마주 비교 메시지에 포함된 정보
발표 시 "실제로 이런 메시지가 매일 아침 옵니다"라고 데모할 수 있는 멘트도 작성해줘.
```

---

## STEP 10. 대시보드 슬라이드

```
Streamlit 대시보드 슬라이드를 만들어줘.
- 4개 탭 구성 (가격 추세, 예측 히스토리, 테마주 비교, 서비스 구조)
- 인터랙티브 Plotly 차트 사용
- 상관관계 히트맵, 수익률 비교표 포함
"웹 개발 없이 Python만으로 이런 대시보드를 만들 수 있다"는 메시지로 작성해줘.
```

---

## STEP 11. 한계점/개선 방향 슬라이드

```
한계점과 개선 방향 슬라이드를 만들어줘.
한계점은 솔직하게, 하지만 "이게 곧 다음 단계의 기회"라는 긍정적 프레임으로:
- 외부 데이터 부재 (뉴스, 금리, VIX)
- Baseline 대비 낮은 성능
- 단일 종목 예측의 한계

개선 방향:
- FinBERT 뉴스 감성분석
- Transformer 계열 시계열 모델
- S&P 500 전체 종목으로 확장
```

---

## STEP 12. 마무리 슬라이드

```
발표 마무리 슬라이드를 만들어줘.
- 프로젝트 전체를 한 문장으로 요약
- 이 프로젝트에서 배운 것 3가지 (데이터 파이프라인, ML 모델링, 서비스화)
- 인상적인 클로징 멘트 (청중이 기억할 수 있는 한 줄)
- Q&A 안내
```

## 보너스: 추가 활용 프롬프트

### 특정 슬라이드 개선
```
[슬라이드 번호]번 슬라이드의 발표 멘트를 더 자연스럽고 설득력 있게 다시 써줘.
청중은 비전공자도 포함되어 있어.
```

### 질문 예상 및 답변 준비
```
이 프로젝트 발표를 들은 교수나 청중이 할 만한 날카로운 질문 5개와
그에 대한 모범 답변을 만들어줘.
```

### 핵심 용어 설명 (비전공자용)
```
이 발표에서 사용된 기술 용어들을 비전공자도 이해할 수 있게
각각 2~3줄로 쉽게 설명해줘:
RSI, MACD, 볼린저밴드, MLPRegressor, FastAPI, n8n, Webhook
```

### 발표 전체 스크립트 생성
```
위 12개 슬라이드를 바탕으로 발표자가 읽을 수 있는
10분 분량의 전체 발표 스크립트를 자연스러운 구어체로 작성해줘.
각 슬라이드 전환 시 자연스러운 연결 멘트도 포함해줘.
```
